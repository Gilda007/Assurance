🔒 SÉCURISATION DU CODE - CHIFFREMENT DES MODULES

🔐 Problématique

Actuellement, le module qui s'installe est constitué d'un code source. Il est nécessaire que le code soit crypté mais lisible uniquement par le logiciel.

💡 Solution proposée : Chiffrement des modules avec AES-256

Mise en place d'un système de chiffrement des modules avant distribution, avec déchiffrement à l'exécution uniquement par l'application.

Architecture proposée

┌─────────────────────────────────────────────────────────────┐ │ SERVEUR DE DISTRIBUTION │ │ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐ │ │ │ Module │ │ Chiffrement │ │ Module │ │ │ │ Source (.py)│ → │ AES-256 │ → │ Chiffré │ │ │ │ │ │ + Clé │ │ (.enc) │ │ │ └─────────────┘ └─────────────┘ └─────────────┘ │ └─────────────────────────────────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────────────────────────────┐ │ APPLICATION CLIENT │ │ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐ │ │ │ Module │ │ Déchiffre- │ │ Module │ │ │ │ Chiffré │ → │ ment avec │ → │ Exécutable │ │ │ │ (.enc) │ │ Clé privée │ │ en mémoire │ │ │ └─────────────┘ └─────────────┘ └─────────────┘ │ └─────────────────────────────────────────────────────────────┘

Mécanisme de chiffrement

# module_encryptor.py - Outil de chiffrement des modules import os import json from cryptography.fernet import Fernet from cryptography.hazmat.primitives import hashes from cryptography.hazmat.primitives.kdf.pbkdf2 import PBKDF2 class ModuleEncryptor: """Chiffrement et déchiffrement des modules""" def __init__(self, master_key): self.master_key = master_key self.kdf = PBKDF2( algorithm=hashes.SHA256(), length=32, salt=b'ams_assurance_salt_2026', iterations=100000, ) self.key = base64.urlsafe_b64encode(self.kdf.derive(master_key.encode())) self.cipher = Fernet(self.key) def encrypt_module(self, module_path, output_path): """Chiffre un module entier""" with open(module_path, 'rb') as f: data = f.read() encrypted = self.cipher.encrypt(data) with open(output_path, 'wb') as f: f.write(encrypted) def decrypt_module(self, encrypted_path): """Déchiffre un module en mémoire (sans écrire sur disque)""" with open(encrypted_path, 'rb') as f: encrypted_data = f.read() return self.cipher.decrypt(encrypted_data)

Chargement sécurisé dans l'application

# secure_loader.py - Chargement des modules chiffrés import importlib.util import sys class SecureModuleLoader: """Chargeur de modules chiffrés""" def __init__(self, decryption_key): self.decryption_key = decryption_key self.encryptor = ModuleEncryptor(decryption_key) def load_encrypted_module(self, module_name, encrypted_path): """Charge un module chiffré en mémoire""" # Déchiffrer en mémoire code = self.encryptor.decrypt_module(encrypted_path) # Créer un module à partir du code déchiffré spec = importlib.util.spec_from_loader(module_name, loader=None) module = importlib.util.module_from_spec(spec) # Exécuter le code déchiffré dans le module exec(code, module.__dict__) # Enregistrer le module sys.modules[module_name] = module return module

Avantages de la solution

  • ✅ Code source non accessible directement par les utilisateurs
  • ✅ Protection contre la copie et la modification non autorisée
  • ✅ Chaque client peut avoir une clé unique
  • ✅ Déchiffrement en mémoire uniquement (pas de trace sur disque)
  • ✅ Compatible avec le système de mise à jour existant
